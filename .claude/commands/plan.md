# Команда: /plan

> Запускает Архитектора для проектирования системы (режим CREATE).

---

## Синтаксис

```bash
/plan
```

---

## Описание

Команда `/plan` создаёт полный архитектурный план для нового MVP проекта.
Используется в режиме CREATE для проектирования системы с нуля.

---

## Агент

**Архитектор** (`.claude/agents/architect.md`)

---

## Режимы

Только **CREATE** — для новых проектов.

Для добавления фичи в существующий проект используйте `/feature-plan`.

---

## Предусловия

| Ворота | Требование |
|--------|------------|
| `PRD_READY` | PRD документ существует |
| `RESEARCH_DONE` | Исследование завершено |

### Алгоритм проверки

```
1. Проверить существование .pipeline-state.json
2. Если файл отсутствует:
   ❌ Пайплайн не инициализирован
   → Сначала выполните /idea
3. Проверить gates.PRD_READY.passed == true
4. Если ворота не пройдены:
   ❌ Ворота PRD_READY не пройдены
   → Сначала выполните /idea
5. Проверить gates.RESEARCH_DONE.passed == true
6. Если ворота не пройдены:
   ❌ Ворота RESEARCH_DONE не пройдены
   → Сначала выполните /research
7. Продолжить выполнение
```

## Выходные артефакты (в целевом проекте)

| Артефакт | Путь |
|----------|------|
| Архитектурный план | `ai-docs/docs/architecture/{name}-plan.md` |

---

## Качественные ворота

### PLAN_APPROVED

| Критерий | Описание |
|----------|----------|
| Компоненты | Все компоненты системы определены |
| API контракты | Эндпоинты и схемы описаны |
| NFR | Нефункциональные требования учтены |
| **Утверждение** | План утверждён пользователем |

**ВАЖНО**: Требуется явное подтверждение от пользователя!

---

## Примеры использования

```bash
# После /research
/plan
```

---

## Следующий шаг

После прохождения ворот `PLAN_APPROVED`:

```bash
/generate
```
